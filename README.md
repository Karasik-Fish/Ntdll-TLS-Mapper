# Manual Mapper (C, No CRT) & ntdll TLS Reverse

Учебный/исследовательский проект. Писал ручной маппер на голом C без CRT. Поддерживает рекурсивную загрузку, резолв импортов, релоки, поиск ApiSets и т.д.

Главная фича — реверс подсистемы TLS в ntdll. Обычно при ручном маппинге TLS крашит деструктор потока при выходе. Я раскопал, как ntdll работает с памятью кучи под TLS:

- В вектор TLS (`GS:[0x58]`) система прячет размер массива по смещению `-0x10`.
- В самих блоках данных DLL прячет сырой указатель из `HeapAlloc` по смещению `-0x8` (его ищет `RtlFreeHeap` при смерти потока).

Чтобы заставить это работать для маппера, написал свой микро-LDE, нашел сигнатуры скрытых переменных (Битмапы, лимиты) и поставил инлайн-хук (14 байт) на неэкспортируемую `LdrpAllocateTls`. Теперь память под скрытые модули выделяется на лету при создании новых потоков.

Пытался сделать рекурсивный маппинг `user32.dll` и `gdi32full.dll`, но они падают глубоко в недрах из-за общения с ядром (`win32k.sys`). Эмулировать ядро из юзермода — отдельная боль, на которую уже не хватило сил. Поэтому системные либы грузятся в начале через обычный `LoadLibrary`, а в идеале надо просто добавить массив исключений в логику загрузки.

Код выкладываю "как есть". Писал 2 месяца, оптимизацию не делал, утечки памяти не чистил, комменты и старые функции оставил как историю своих страданий. Тестовые DLL для проверки функционала тоже в репе.
